# DatePicker

Masked date field with a **popover calendar**, **Luxon** parsing, and **react-hook-form** integration. The form value is the **formatted string**; use **`onChange`** for a **`Date`**. Pick **day**, **month**, or **year** mode with **`pickerType`**, choose **`openCalendarMode`** for icon vs input vs both, and set **`dateMin`** / **`dateMax`** to constrain the range. Wrap the field in **`FormProvider`** (see stories decorator).

## Default Usage

The DatePicker component provides an intuitive interface for selecting dates with input masking and calendar popup. Here's how to use it:

```tsx
import { DatePicker } from '@ssa-ui-kit/core';

function BasicDatePicker() {
  const handleDateChange = (date?: Date) => {
    console.log('Selected date:', date);
  };

  return (
    <DatePicker
      name="birthdate"
      label="Birth Date"
      helperText="Select your birth date"
      onChange={handleDateChange}
    />
  );
}
```

## Features

### Date Formats

Support for different date formats with automatic input masking:

```tsx
// US format (default)
<DatePicker
  name="date-us"
  label="US Format"
  format="mm/dd/yyyy"
  onChange={handleDateChange}
/>

// European format
<DatePicker
  name="date-eu"
  label="European Format"
  format="dd/mm/yyyy"
  onChange={handleDateChange}
/>
```

### Calendar Opening Modes

Control how users can access the calendar popup:

```tsx
// Icon only (default)
<DatePicker
  name="date-icon"
  label="Icon Only"
  openCalendarMode="icon"
  onChange={handleDateChange}
/>

// Input field click
<DatePicker
  name="date-input"
  label="Input Click"
  openCalendarMode="input"
  onChange={handleDateChange}
/>

// Both icon and input
<DatePicker
  name="date-both"
  label="Icon and Input"
  openCalendarMode="both"
  onChange={handleDateChange}
/>
```

### Date Range Restrictions

Set minimum and maximum date boundaries:

```tsx
<DatePicker
  name="restricted-date"
  label="Date Range"
  dateMin="01/01/2024"
  dateMax="12/31/2025"
  onChange={handleDateChange}
/>
```

### Default Values

Set initial dates for the picker:

```tsx
<DatePicker
  name="preset-date"
  label="With Default"
  defaultValue="02/10/2025"
  onChange={handleDateChange}
/>
```

### External Value Control

Control the date value externally for programmatic updates:

```tsx
import { useState, useEffect } from 'react';

function ControlledDatePicker() {
  const [selectedDate, setSelectedDate] = useState('01/15/2025');

  useEffect(() => {
    // Programmatically update date
    const timer = setTimeout(() => {
      setSelectedDate('02/10/2025');
    }, 3000);
    return () => clearTimeout(timer);
  }, []);

  return (
    <DatePicker
      name="controlled-date"
      label="Controlled Date"
      value={selectedDate}
      onChange={(date) => {
        if (date) {
          setSelectedDate(date.toLocaleDateString('en-US'));
        }
      }}
    />
  );
}
```

### Time Selection

Set **`showTimePicker`** to add hours and minutes columns beside the calendar. The
field's format and mask gain a **` HH:mm`** suffix, and the emitted **`Date`** keeps its
time instead of being floored to midnight:

```tsx
<DatePicker
  name="meeting"
  label="Meeting starts"
  showTimePicker
  defaultValue="01/15/2025 09:30"
  onChange={(date) => console.log(date.getHours(), date.getMinutes())}
/>
```

**`minuteStep`** controls the minutes column and defaults to **15**. Use **`1`** for the
full 0–59 list:

```tsx
<DatePicker name="precise" showTimePicker minuteStep={1} />
```

Two behaviours change when time is enabled:

- Selecting a day **does not close** the popover — otherwise the time columns would be
  unreachable. It closes on outside click instead.
- Stored values must include the time (**`01/15/2025 09:30`**). A bare **`01/15/2025`** no
  longer matches the mask and will not parse.

### Clear Button

Set **`showClearButton`** to add a **×** beside the calendar icon. It appears only while the
value differs from **`defaultValue`** and restores that value when clicked — so with no
**`defaultValue`** it simply empties the field:

```tsx
<DatePicker
  name="due"
  label="Due date"
  showClearButton
  defaultValue="03/20/2025"
/>
```

### Success State

Set **`success`** to render the field in its success state. A validation error always wins,
so a field with an error is never shown as successful:

```tsx
<DatePicker
  name="confirmed"
  label="Confirmed date"
  success
  helperText="Date confirmed"
/>
```

### Disabled State

Disable the date picker when needed:

```tsx
<DatePicker
  name="disabled-date"
  label="Disabled Picker"
  disabled={true}
  defaultValue="02/10/2025"
  onChange={handleDateChange}
/>
```

### Event Handling

Comprehensive event handling for different interactions:

```tsx
<DatePicker
  name="event-date"
  label="Event Handling"
  onChange={(date) => console.log('Date changed:', date)}
  onOpen={() => console.log('Calendar opened')}
  onClose={() => console.log('Calendar closed')}
  onError={(date, error) => console.log('Error:', error)}
  onMonthChange={(date) => console.log('Month changed:', date)}
  onYearChange={(date) => console.log('Year changed:', date)}
  onBlur={(event) => console.log('Input blurred')}
/>
```

### Input Reference

Access the input element directly using refs:

```tsx
import { useRef } from 'react';

function DatePickerWithRef() {
  const inputRef = useRef<HTMLInputElement>(null);

  const focusInput = () => {
    inputRef.current?.focus();
  };

  return (
    <div>
      <DatePicker
        ref={inputRef}
        name="ref-date"
        label="Date with Ref"
        defaultValue="02/10/2025"
        onChange={handleDateChange}
      />
      <button onClick={focusInput}>Focus Input</button>
    </div>
  );
}
```

### Custom Styling

Apply custom CSS classes to different parts of the component:

```tsx
<DatePicker
  name="styled-date"
  label="Custom Styled"
  onChange={handleDateChange}
  classNames={{
    header: 'custom-header',
    trigger: {
      input: 'custom-input',
      calendarIcon: 'custom-icon',
    },
    monthsSwitch: {
      wrapper: 'custom-month-wrapper',
      previousMonth: 'custom-prev',
      nextMonth: 'custom-next',
    },
    calendar: 'custom-calendar',
    label: 'custom-label',
  }}
/>
```
